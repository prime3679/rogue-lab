# REVIEW

- Start with defects, regressions, missing verification, and safety boundary violations.
- Classify `wrangler.toml`, bindings, routes, migrations, cron behavior, auth, and cloud mutation risk as infrastructure review, not cosmetic review.
- Require exact local evidence for any claim that a fresh agent can verify a change.
- Escalate when the repo cannot prove a claim without installs, secrets, remote access, or data mutation.
